# Landing Backend

REST API для обработки заявок с лендинга с использованием искусственного интеллекта для автоматической классификации обращений.

Приложение принимает заявки пользователей, определяет их категорию с помощью Google Gemini, сохраняет данные в базу, отправляет уведомления по электронной почте (при наличии SMTP-конфигурации) и предоставляет сервисные метрики.

---

# Основные возможности

- REST API на FastAPI
- Валидация входящих данных
- Классификация обращений с помощью Google Gemini
- Хранение данных в SQLite
- Архитектура Repository + Service
- Отправка уведомлений по электронной почте
- Ограничение количества запросов (Rate Limiting)
- Логирование работы приложения
- Health Check endpoint
- Metrics endpoint
- Swagger UI

# Установка

Клонируйте репозиторий:

```bash
git clone https://github.com/afkXesP/landing-backend.git

cd landing-backend
```

Создайте виртуальное окружение:

```bash
python -m venv .venv
```

Активируйте его.

Windows:

```bash
.venv\Scripts\activate
```

Linux/macOS:

```bash
source .venv/bin/activate
```

Установите зависимости:

```bash
pip install -r requirements.txt
```

---

# Конфигурация

Создайте файл `.env`.
```bash
cp .env_example .env
```

Пример:

```env
APP_NAME=Landing Backend API
APP_VERSION=1.0.0

DEBUG=True

HOST=127.0.0.1
PORT=8000

DATABASE_URL=sqlite:///storage/database.db

GOOGLE_API_KEY=YOUR_API_KEY
GEMINI_MODEL=gemini-3.5-flash

SMTP_HOST=
SMTP_PORT=587
SMTP_USERNAME=
SMTP_PASSWORD=
SMTP_FROM=

OWNER_EMAIL=owner@example.com

RATE_LIMIT=5/minute
```

# API

## Создание заявки

**POST**

```
/api/contact
```

Пример запроса:

```json
{
  "name": "John Smith",
  "email": "john.smith@example.com",
  "phone": "+1 555 123 4567",
  "message": "I'd like to know the pricing of your services."
}
```

Пример ответа:

```json
{
  "status": "success",
  "message": "Request successfully received.",
  "request_id": 1
}
```

# Использование искусственного интеллекта

Для классификации входящих обращений используется Google Gemini.

Модель определяет одну из категорий:

- sales
- support
- other

Если AI-сервис временно недоступен или возвращает некорректный результат, приложение автоматически использует значение:

```
unknown
```

Это позволяет API продолжать работу даже при недоступности внешнего сервиса.

---
# Используемый промпт

Для классификации входящих обращений используется следующий промпт:

```text
You are a request classifier.

Classify the user's request into exactly one category.

Rules:
- Return exactly one word.
- Do not explain.
- Do not use punctuation.

Allowed values:
- sales
- support
- other

Message:
{message}
```

Такой формат ответа позволяет минимизировать количество неоднозначных результатов и упростить последующую обработку ответа модели.


### Почему именно такой промпт

При проектировании промпта основной целью было получить максимально предсказуемый ответ от модели.

Поэтому были использованы следующие ограничения:

- ответ должен состоять ровно из одного слова;
- запрещены любые пояснения;
- запрещены знаки препинания;
- список допустимых категорий ограничен тремя значениями (`sales`, `support`, `other`).

Благодаря этому ответ модели можно напрямую преобразовать в значение перечисления (`RequestCategory`) без дополнительной обработки текста.

# Уведомления по электронной почте

После успешного создания заявки приложение пытается отправить:

- уведомление владельцу сайта;
- письмо-подтверждение пользователю.

Если SMTP не настроен, отправка писем пропускается, а в журнал приложения записывается предупреждение.

При этом заявка успешно сохраняется в базе данных.

# Ограничение запросов

Для защиты API используется **SlowAPI**.

По умолчанию установлено ограничение:

```
5/minute
```

При необходимости значение можно изменить через переменную окружения `RATE_LIMIT`.

# Архитектура

Проект построен с использованием многослойной архитектуры.

```
      Клиент
        │
        ▼
       API
        │
        ▼
   Service Layer
        │
        ▼
  Repository Layer
        │
        ▼
    База данных
```

Такое разделение позволяет изолировать бизнес-логику от HTTP-слоя и слоя доступа к данным, упрощая сопровождение и тестирование приложения.
